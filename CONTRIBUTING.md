# Contributing

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.

## How to Contribute

1. Fork the repository
2. Create a new branch:
   'git checkout -b feature/your-feature-name'
3. Make your changes and commit them:
    'git commit -m "Add your message here"'
4. Push to the branch:
    'git push origin feature/your-feature-name'
5. Open a **Pull Request**

## Reporting Bugs

- Check if the bug has already been reported in the [Issues](../../issues) tab
- If not, open a new issue with a clear title and description
- Include steps to reproduce the bug

## Bug Fixes

- Fork the repo and create a branch named `bug-fix-<issue-number>`
- Fix the bug and submit a Pull Request referencing the issue

## Documentation Improvements

- Spotted a typo or unclear explanation? Feel free to open a PR with improvements
- Good documentation is just as important as good code

## Enhancements

- Open an issue with the tag `enhancement` before starting work
- Describe the feature and why it would be useful

## Ideas

- Have an idea but not sure how to implement it? Open a discussion or issue
- All ideas are welcome and will be considered

## Code of Conduct

Please read our [Code of Conduct](CODE_OF_CONDUCT.md) before contributing.
